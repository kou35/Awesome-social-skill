## What Is This 🧠

`Digital souls` 指的是那些可以被记录、提炼、迁移和复用的人类痕迹: 说话方式、判断习惯、职业经验、情感回应、记忆碎片、关系角色，甚至是某种非常具体的“这个人会怎么做”的感觉。它不是把人简化成数据，而是试图为人的经验留下一个可交互、可延续、可被重新调用的接口。

`Skills distillation` 指的是把原本只存在于某个人身上的能力，拆解并沉淀成可以复用的 AI 资产，例如 `skill`、`agent`、`persona`、workflow 或 memory system。它关心的不是“复制一个万能人类”，而是把那些真正有价值的经验单位提炼出来: 怎么沟通，怎么安慰人，怎么写周报，怎么带学生，怎么在深夜救一个快崩溃的实验。

围绕大模型与技能蒸馏，常常同时存在两种情绪：

> “你们搞大模型的就是码奸，你们已经害死前端兄弟了，还要害死后端兄弟，测试兄弟，运维兄弟，害死网安兄弟，害死ic兄弟，最后害死自己害死全人类”

> “你们搞大模型的简直是码神，你们解放了前端兄弟，还要解放后端兄弟，测试兄弟，运维兄弟，解放网安兄弟，解放ic兄弟，最后解放自己解放全人类”

这个仓库更愿意正面承认这种张力，但立场也很明确：我们是**人类中心主义**的。技术的目标不是替人类的存在让路，而是为人类服务；不是抹平人的独特性，而是保存、放大、传递那些本来会随着时间流失的经验、人格与连接。

所以这里收集的，不只是项目链接，也是一组问题：

- 一个人的经验，能被蒸馏到什么程度
- 一段关系的陪伴，能以什么形式被延续
- 一个角色的判断与风格，怎样才能被尊重地保存下来
- 当我们构建 `digital souls` 时，怎样始终把“人”放在中心

## How To Contribute 🤝

欢迎补充更多“数字永生”方向的 skill 设定或开源项目。建议优先收录：

- 有明确角色设定与使用场景的 `skill` / `agent`
- 能体现人格、记忆、职业经验沉淀的项目
- 可以复现、可追踪来源、最好附带仓库或 Demo 的内容

建议提交格式：

```md
| `skill-name` | 角色定位 | 适用场景 | 核心能力 |
```
